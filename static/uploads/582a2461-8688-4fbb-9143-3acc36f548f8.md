### **주식 분석 프로그램 개발 요구사항 명세서 (전문가 Ver.)**

#### **1. 핵심 철학: "데이터를 정보로, 정보를 통찰력으로"**
가장 중요한 것은 단순히 데이터를 나열하는 것이 아니라, 사용자가 '그래서 지금 사야 하는가, 팔아야 하는가, 기다려야 하는가?'에 대한 **의사결정을 내릴 수 있도록 돕는 것**입니다. 모든 기능은 이 철학을 중심으로 설계되어야 합니다.

**Tier 1: 가장 기본적인 핵심 기능 (MVP)**

*   **데이터 수집 및 관리 (Data Acquisition & Management)**
    *   **필수 데이터:**
        *   **시세 데이터 (OHLCV):** 시가(Open), 고가(High), 저가(Low), 종가(Close), 거래량(Volume). 최소 일봉, 분봉(1, 5, 10, 30, 60분) 데이터 확보가 필수적입니다.
        *   **기업 기본 정보:** 종목명, 종목코드, 시가총액, 상장주식수, 소속 시장(코스피/코스닥) 등.
    *   **데이터 소스:** 국내 증권사 API (한국투자증권, 키움증권 등) 또는 해외 데이터 제공 API (Alpha Vantage, Polygon.io 등)를 활용한 안정적인 데이터 수급 파이프라인 구축.
    *   **데이터 저장:** 수집한 데이터를 효율적으로 저장하고 빠르게 조회할 수 있는 데이터베이스(DB) 설계 (e.g., Time-series DB인 InfluxDB 또는 RDBMS).

*   **데이터 시각화 (Data Visualization)**
    *   **인터랙티브 차트:** 사용자가 기간을 조절하고, 확대/축소하며 데이터를 탐색할 수 있는 동적 차트. (Candlestick 차트가 기본)
    *   **보조지표 표시:** 차트 위에 이동평균선(SMA, EMA), 볼린저 밴드 등 기본 보조지표를 중첩해서 표시할 수 있는 기능.
    *   **거래량 차트:** 가격 차트 하단에 거래량 바 차트 표시.

*   **기본 기술적 분석 (Technical Analysis)**
    *   **이동평균선 (Moving Average):** 5, 20, 60, 120일선 등 사용자가 정의한 기간의 이동평균선을 계산하고 차트에 표시.
    *   **핵심 보조지표:**
        *   **RSI (Relative Strength Index):** 과매수/과매도 구간 판단.
        *   **MACD (Moving Average Convergence Divergence):** 추세의 강도와 방향성 판단.
        *   **Stochastic Oscillator:** 현재 주가 위치 판단.

**Tier 2: 경쟁력을 갖추기 위한 심화 기능**

*   **기본적 분석 (Fundamental Analysis)**
    *   **재무 데이터 연동:** 분기/연간 재무제표(매출, 영업이익, 당기순이익) 데이터 연동 및 시각화.
    *   **핵심 가치 지표(Value Metrics) 계산:**
        *   **PER (주가수익비율)**
        *   **PBR (주가순자산비율)**
        *   **ROE (자기자본이익률)**
        *   **EPS (주당순이익)**
        *   **배당수익률**

*   **종목 스크리닝 (Stock Screening)**
    *   사용자가 원하는 조건으로 종목을 검색하는 기능.
    *   **예시:** "PER 10 이하 AND ROE 15 이상 AND 현재 주가가 20일 이동평균선 위에 있는 종목 찾아줘."
    *   기술적 지표(e.g., RSI 30 이하)와 기본적 지표(e.g., PBR 1 이하)를 조합한 조건 검색 기능.

*   **백테스팅 (Backtesting)**
    *   **가장 중요한 기능 중 하나.** 사용자가 자신의 매매 전략이 과거에 유효했는지 검증하는 기능.
    *   **필수 요소:**
        *   **진입/청산 조건 설정:** "20일선이 60일선을 상향 돌파(골든크로스)하면 매수, 하향 돌파(데드크로스)하면 매도" 와 같은 전략 설정.
        *   **백테스팅 기간 설정:** 특정 과거 기간 동안 시뮬레이션.
        *   **결과 리포트:** 총수익률, 연평균수익률(CAGR), 승률, 최대 낙폭(MDD) 등 성과 지표 제공.

**Tier 3: 전문가 수준의 고급 기능**

*   **실시간 알림 (Real-time Alerts)**
    *   사용자가 설정한 조건 도달 시 알림 기능 (이메일, SMS, 푸시 알림 등).
    *   **예시:** "삼성전자 주가가 8만원 도달 시 알림", "SK하이닉스 RSI가 70을 넘으면 알림".

*   **시장 분석 대시보드 (Market Analysis Dashboard)**
    *   **시장 전체 현황:** 상승/하락 종목 수, 코스피/코스닥 지수, 투자자별(개인/기관/외국인) 순매수 동향.
    *   **주도 업종/테마 분석:** 당일 가장 강한 흐름을 보이는 업종이나 테마를 시각적으로 표시.

*   **뉴스 및 공시 데이터 연동 (News & Disclosure Integration)**
    *   특정 종목의 뉴스, 공시를 수집하여 차트와 함께 타임라인에 표시.
    *   (고급) 자연어 처리(NLP)를 이용한 뉴스 감성 분석 (긍정/부정 뉴스 분류).

#### **3. 비기능적 요구사항 (Non-Functional Requirements)**

*   **데이터 정확성 (Data Accuracy):** 데이터가 틀리면 모든 분석은 의미가 없습니다. 데이터 정합성을 최우선으로 확보해야 합니다.
*   **성능 (Performance):** 차트 로딩, 백테스팅 실행 속도가 느리면 사용자는 떠납니다. 특히 실시간 데이터 처리는 빠른 응답 속도가 생명입니다.
*   **보안 (Security):** 사용자의 API 키나 개인 설정 등 민감한 정보는 안전하게 암호화하여 관리해야 합니다.
*   **사용자 경험 (UX/UI):** 복잡한 금융 정보를 초보자도 쉽게 이해하고 사용할 수 있도록 직관적이고 깔끔한 인터페이스를 제공해야 합니다.

#### **4. 개발 로드맵 제안**

*   **Phase 1 (MVP):** Tier 1 기능에 집중합니다. 한 종목의 시세 데이터를 가져와 차트를 그리고, 기본 보조지표 2~3개를 표시하는 것부터 시작하세요.
*   **Phase 2 (확장):** Tier 2 기능인 기본적 분석과 스크리닝 기능을 추가하여 사용성을 높입니다.
*   **Phase 3 (고도화):** 백테스팅 엔진을 정교하게 만들고, 알림 및 자동화 기능을 추가하여 전문가용 툴로 발전시킵니다.
